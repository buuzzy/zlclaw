# v1.0.1 — 首轮内测修复

> 2026-04-23

首轮内测前的稳定性修复 + 一次品牌重命名。建议所有 v1.0.0 用户通过"设置 → 关于 → 检查更新"自动升级。

## 修复

- **修复 session 标题被 `<think>...</think>` 污染**：MiniMax-M2.5-thinking 等推理模型会把内部推理混进 content，直接被保存成 task.prompt。前后端双层 sanitize。
- **修复 StockSnapshot 白屏**：当后端返回的 mktCap / turnover / turnoverRate / pe / pb / roe 任一字段为 undefined 时，`.toFixed()` / `.toLocaleString()` 直接抛错，冒泡到 router 错误边界整页崩溃。加 null-safe 守卫，缺失字段统一渲染为 `—`。

## 功能

- **M1 — 本地数据按账号隔离**：原本 `~/Library/Application Support/ai.sage.desktop/sage.db` 是全局共享的，用户 A 登录创建的会话在用户 B 登录后仍可见，存在严重隐私问题。现改为 `~/.sage/users/{uid}/sage.db` + `~/.sage/users/{uid}/sessions/`，按 Supabase uid 物理隔离；旧版数据自动一次性迁移到首个登录用户的目录（复制，不移动，保留回退 safety）。
- **M2 — App 内检查更新**：设置 → 关于加"检查更新"按钮。基于 `tauri-plugin-updater` + Ed25519 签名。
- **M3 — 更新红点提示**：启动 3s 后静默检查更新，有新版时 sidebar 头像 / 设置菜单 / "关于" Tab 出现红点，打开"关于"自动消除。
- **M4a — Supabase 环境分离（代码层）**：`supabase.ts` 改为读取 `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY` 环境变量，dev / prod 可分离。AboutSettings 底部显示当前环境标签。

## 体验

- **品牌重命名**：Sage → 涨乐金融龙虾。新 logo + 作者 nakocai + 底标 "Based on WorkAny by nakocai"。底层 `~/.sage/` / `sage://` / `ai.sage.desktop` 等标识符保持不变。
- **同步状态 UI 重构**：删除 sidebar 底部的文字状态栏（在正常态频繁闪烁干扰）。改为 macOS native 风格——头像左上角静止绿点（ok）/ 无装饰（syncing）/ 半透明遮罩 + WifiOff 图标（failed，点击立即重试）。hover 头像显示详细 tooltip。

## 技术债记录

- Tauri fs scope glob `**` 不匹配 `.` 开头的隐藏文件，已在 capabilities 额外加 `$HOME/.sage/.*` 规则
- Updater 签名密钥 v1（空密码）作废，v2 用非空密码重新生成，pubkey 已更新到 `tauri.conf.json`
- `latest.json` 和 release 流程已固化在 `docs/RELEASE.md`

## 升级说明

- **自动升级**：v1.0.0 用户打开 app → 设置 → 关于 → 检查更新（或 sidebar 头像红点提示）
- **手动安装**：下载本 release 的 DMG，拖入 Applications 覆盖旧版
- 本地数据兼容：升级过程中 `~/Library/.../sage.db` 会一次性迁移到 `~/.sage/users/{你的 uid}/sage.db`，会话全部保留
