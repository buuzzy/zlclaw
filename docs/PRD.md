> 版本：v0.1.5
> 日期：2026-04-20

---

## 1. 产品概述

### 1.1 产品简介

**HT Claw** 是一款垂直金融领域桌面 AI Agent，通过自然语言交互，为个人投资者提供金融数据查询、行情可视化、资讯追踪和 AI 辅助研究功能。

### 1.2 核心价值

| 价值点 | 描述 |
|--------|------|
| **数据全面** | 11 个金融技能覆盖行情/财务/基金/研报/公告/宏观/行业，A 股/港股/美股全覆盖 |
| **可视化强** | K 线图表（TradingView 级）、柱状图、折线图、数据表格、行情卡片、新闻列表、热力图、研报评级、财务健康仪表盘等 13 种组件 |
| **多渠道** | 桌面端 + 微信 + 飞书，随时随地获取金融数据 |
| **有记忆** | 记住用户偏好和关注标的，每日自动归纳，个性化回复 |
| **定时推送** | Cron 定时任务，财经早餐、价格预警自动推送 |
| **开箱即用** | API Key 预置，下载即用 |

### 1.3 目标用户

**主要用户**：有基本金融知识的中国个人投资者（A 股/港股/美股）

**用户特征**：
- 有投资理财需求，关注市场动态
- 希望快速获取行情、财务数据、研报等信息
- 愿意用自然语言与工具交互
- 对 AI Agent 有基本认知

## 2. 用户场景

### 场景 1：查询实时行情

**用户**：腾讯今天涨了多少

**系统**：
1. 识别意图 → 调用「行情数据查询」技能
2. 输出文字摘要 + `artifact:quote-card` 行情卡片
3. 卡片展示：代码、名称、现价、涨跌幅、开高低、成交量

### 场景 2：查看 K 线走势

**用户**：给我看苹果最近一个月的日 K 线

**系统**：
1. 识别意图 → 调用「行情数据查询」技能获取历史数据
2. 输出 `artifact:kline-chart`，前端渲染 TradingView K 线图表
3. 支持日 K / 周 K / 月 K 切换

### 场景 3：财务数据分析

**用户**：茅台近三年营收和净利润

**系统**：
1. 调用「财务数据查询」技能
2. 输出 `artifact:data-table` 数据表格，或 `artifact:bar-chart` 柱状图对比

### 场景 4：搜索研报

**用户**：找几篇关于新能源汽车的研报

**系统**：
1. 调用「研报搜索」技能
2. 输出 `artifact:data-table` 研报列表（标题、机构、日期、评级）

### 场景 5：财经资讯

**用户**：今天有什么 AI 相关的新闻

**系统**：
1. 调用「新闻搜索」技能
2. 输出 `artifact:news-list` 新闻列表

### 场景 6：宏观数据

**用户**：最近几个月的 CPI 数据

**系统**：
1. 调用「宏观数据查询」技能
2. 输出 `artifact:line-chart` 折线图展示趋势

### 场景 7：多渠道使用

**用户**（通过微信）：上证指数今天怎么样

**系统**：
1. WeClaw 转发到 `/v1/chat/completions` 端点
2. Agent 查询数据，返回纯文本摘要（渠道端不渲染可视化组件）
3. 支持会话连续性（30 分钟窗口内追问）

### 场景 8：记忆与个性化

**用户**：我最近在关注半导体板块

**系统**：
1. 记录到 `~/.htclaw/MEMORY.md`
2. 后续对话中，Agent 自动关联半导体相关信息
3. 每日 23:00 自动归纳对话要点，更新长期记忆
4. 用户说「我关注的板块怎么样」时，Agent 知道指半导体

### 场景 9：定时任务

**用户**：帮我设置一个每天早上 8:30 推送财经早餐的任务

**系统**：
1. Agent 调用 Cron 工具创建定时任务
2. 每天 8:30 自动查询市场动态并推送到当前渠道
3. 用户可在设置面板查看/删除任务

### 场景 10：个股综合快照

**用户**：帮我看下宁德时代的完整行情，包括估值指标

**系统**：
1. 调用「行情数据查询」+ 「财务数据查询」技能
2. 输出 `artifact:stock-snapshot`，含大字价格、迷你 sparkline、估值三表、分析师目标价

### 场景 11：板块热力图

**用户**：今天各板块涨跌情况怎么样，用热力图展示

**系统**：
1. 调用「行业数据查询」技能
2. 输出 `artifact:sector-heatmap`，ECharts Treemap，颜色 = 涨跌幅，面积 = 成交额

### 场景 12：研报评级汇总

**用户**：比亚迪最近的分析师评级怎么样

**系统**：
1. 调用「研报搜索」技能
2. 输出 `artifact:research-consensus`，含评级分布横条、目标价区间、研报列表

### 场景 13：财务健康分析

**用户**：帮我分析下阿里巴巴的财务健康状况

**系统**：
1. 调用「财务数据查询」+ 「公司经营数据查询」技能
2. 输出 `artifact:financial-health`，4 维度卡片 2×2 网格 + 一句话摘要

### 场景 14：情绪新闻流

**用户**：帮我搜下最近关于 AI 芯片的资讯，带情绪分析

**系统**：
1. 调用「新闻搜索」技能
2. 输出 `artifact:news-feed`，Timeline 风格，每条含情绪圆点（牛市绿/熊市红/中性灰）

## 6. 竞品对比

| 竞品 | 优势 | 劣势 | HT Claw 差异 |
|------|------|------|-------------|
| 同花顺 | 数据全、用户多 | 臃肿、AI 能力弱 | 自然语言交互 + 轻量桌面端 |
| 东方财富 | 资讯强、社区活跃 | 交互复杂 | 对话式获取信息 |
| 富途牛牛 | 港美股体验好 | AI 功能有限 | Agent 驱动 + 多渠道 |
| 通义/文心等通用 AI | 通用能力强 | 无专业金融数据 | 垂直金融技能 + 专业可视化 |

## 7. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| iwencai API 不可用 | 金融数据查询失败 | 错误降级到 WebSearch，纯文本回复 |
| API Key 泄露 | 数据安全 | Key 存于本地 config.json，不入版本控制 |
| LLM 幻觉 | 编造金融数据 | AGENTS.md 严格约束不编造 + 数据必须来自技能调用 |
| 跨平台兼容 | 特定平台 bug | CI 四平台构建 + 定期测试 |
| 渠道消息重复/丢失 | 用户体验差 | 持久化去重缓存 + 消息补偿机制 + 速率限制 |
